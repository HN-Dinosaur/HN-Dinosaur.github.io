windows下通过Github + Hexo的Blog创建

Under Windows build Hexo environment

Estimated Reading Time 2 Minutes
Words 721 In Total
Viewed Times

注册Github账号

安装Git

Git

New repository

在自己的GitHub上New一个repository

Repository name 必须是GitHub用户名.github.io

创建SSH Key

生成SSH Key

打开Git Bash命令窗口输入,两次回车后会生成id_rsa,id_rsa.pub两个文件

选择主题

从repository的Setting进入 点击上图的Choose a theme

进入下一个页面点击Commit changes

添加SSH到Github

复制id_rsa.pub文件里的内容（全部复制），打开GitHub-Settings-SSH and GPG keys,添加SSH keys

创建本地仓库

打开之前创建的repository，点击Code，复制SSH的地址

在自己的电脑硬盘随便一个位置创建一个文件夹，用来存放本地的repository

在自己的电脑本地的repository的文件夹下，鼠标右键点击空白处，选择Git Bush 输入 Git Clone + SSH的地址，将仓库内容复制到本地

从仓库直接上传文件

在repository根目录下写一个简单的html文件,右键空白出打开Git Bush输入

直接上传此html文件

安装Node.js（建议下载二进制）

Node.js

安装Hexo

Hexo

后面的步骤的意思大致是以Hexo为框架搭建，最后展示形式在Github Pages上边

本地Blog文件

再在你的电脑上创建一个文件夹存放Blog文件，并在此文件夹空白处右键打开Git Bush输入hexo init生成如下文件

发布文章

从本地blog文件右键空白处打开Git Bush输入

可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。根据不同的布局，会在不同的路径下生成title.md文件。

title是文章的标题，tags是标签，还可以添加categories分类，对发表的博文进行归纳整理。
—分割线之后的部分就是文章的内容。可用Markdown语法自由编辑。

配置部署Hexo

安装hexo-deployer-git

直接打开Git Bush

修改_config.yml

修改你电脑本地blog文件中_config.yml文档中deploy的配置指定位置如下图

type: git # git方式部署

repo: # repository地址，即GitHub用户名.github.io库的SSH地址

branch: [branch] #分支名字，master即可

message: [message] #自定义提交信息时间（默认为当前时间）

完成部署

本地blog右键空白处

打开Git Bush输入完成部署

上传第一个MarkDown文件

本地Blog文件夹右键打开Git Bush

http://localhost:4000/. 预览页面

生成静态文件

最后一步

打开GitHub删除main分支稍等5min，再输入 你的GitHub用户名.github.io 为网址即可访问

Have fun ^_^

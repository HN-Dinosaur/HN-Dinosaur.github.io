JAVA快速两个文件中的词-将相同词和词向量存储

目的

大文件–16G腾讯做自然语言处理的文件

小文件–随便一个不到1G的词云文件

大文件的内容大致是每一行开头一个词+词向量 空格相隔

小文件就是[“xx”,”yiy”]词的数组

将大文件的内容和小文件中的词进行比较，如果相同则将大文件的词和词向量记录，最后总的进行存储入另外一个文件中

思路

1.读取

2.比较

3.存储

大致就是以上几个步骤了，但是实际做起来得费不少劲，我从2021-4-30下午开始阅读题目，直到2021-5-3早晨才彻底解决

遇到的问题

1.JVM中out of heap

我一开始想的是，将大文件中的数据读取出来，存储在HashMap里面，但是在取出来的过程中，程序报错，经过查询，是超过JVM里的内存大小了，我想着扩大JVM内存，最大就只能改到4G，果断放弃这个方法。

2.怎么快速读取

我通过搜索，我找到网络上有几个方法，有I/O还有NIO读取，经过试验，大致基本都相同。这里我用的是BufferedReader来进行大文件的一行一行读取

3.怎么快速存储

拿的HashMap存储结构，还有ArrayList，StringBuilder，HashMap比较最快，ArrayList遍历最快，StringBuilder做字符串存储最快

4.单线程还是多线程进行操作？

这里我尝试过TheadPool来多线程处理，可是电脑给跑关机了…

5.字符串处理

两种方式处理吧，第一个是正则表达式，第二个是内置的字符串方法

6.什么遍历方式

7.局部变量和全局变量

局部变量和全局变量同名闹了一些乌龙，费了半天的劲，突然看到变量的颜色不同才发现问题，建议大家在进行变量命名时，第一命名有意义，第二全局变量前面加一个_

8.序列化和非序列化存储

一开始以为序列化存储会对存储的时间进行压缩，最后发现存储的时间压根不是主要，比较才是主要，最后不用序列化，拿的是BufferWritter进行文件的存储

9.比较相等时的存储字符串类型

如果拿String来进行结果的存储的话，基本类型String的每次操作都是重新构建String数据类型，这样的话就会很慢，我这里拿的是StringBuilder来进行存储

结果

目前的结果是91s，希望未来有机会对这个项目进行改进

改进

2.网络
